## 逐筆短評

### case_00（會員 App 點餐自取）
兩模型內容皆完整覆蓋（金流取捨、QR 核銷、取餐時限、試營運）。共同問題是**捏造期限**：逐字稿中淑芳只說「跟幾個店長評估一下再回報」（無期限，參考摘要作「未定」），base 寫「週內完成評估並回報」、tuned 寫「本週」；秀琴挑試營運店一項兩者也都補上了逐字稿沒有的「週內／本週」。tuned 決議段「未定時限、退款政策……」語句不通順。

### case_01（重複扣款緊急應變）
base 覆蓋極完整（負餘額 30 餘戶優先沖正、簡訊去重、媒體口徑進行式等全有），但檔尾殘留截斷亂碼「`| �`」（第 36 行）。tuned 有兩處**負責人張冠李戴**：「完成沖正退款」「產出沖正清單」都掛給怡萱，逐字稿實為明哲請帳務執行、明哲產清單（「去重後的名單我產沖正清單的時候一起給妳」）；且**整份輸出完全沒提「中午前發簡訊與 App/官網公告」**這個核心行動項目。此筆 tuned 明顯落後。

### case_02（年度募款分工）
兩模型皆把「曉琪」寫成簡體「**晓**琪」，且都漏了「佩玲提供社群推廣素材與話術」這個行動項目（參考摘要有列）。base 有錯字「晉升線上模式」；tuned 有錯字「晚年實體園遊會」（應為「去年」）。其餘三個行動項目兩者的事項／負責人／期限均正確。

### case_03（快省 Pass 訂閱免運）
數字面（99 月費、45 元運費、2.2 單、首月 49）兩者皆正確。共同錯誤：把「系統規格」的提供者寫成阿廷（逐字稿是阿廷「收到」規格才能開工，提供方不是他），base 的「完成系統規格書並提交給阿廷團隊｜阿廷」自相矛盾；兩者也都漏了「家豪拿企劃去跟財務對」。base 另把「fair use 判斷邏輯三天做完」寫成「三日內完成測試環境部署」，屬輕度扭曲；tuned 有「賺本」「一倍五」等不通順措辭。

### case_04（供應商交期與 8D 品質）
base 覆蓋最完整（含出貨附 bow and twist 實測數據、上季調價 4%），三個行動項目與參考完全一致；但把 safety stock「台灣倉備**一個月**的量」寫成「備每月**一週**量」，數字錯誤。tuned 問題較多：把買方的「月需求兩千五到三千二」誤寫成「**供應商**月需求」、把 Kevin 主動提的 safety stock program 寫成「美玲要求」、8D 補齊 D5–D8 的負責人誤植為 **Kevin**（應為 Sandy），且**完全遺漏「每批出貨附實測數據、檢驗報告改版」這項決議**。

### case_05（線上信貸 KYC 對齊）

## 結論

**平均分對比**（base → tuned）：

- 忠實度：4.2 → 4.2（持平）
- 要點覆蓋：4.9 → 4.2（**退步 0.7**）
- 行動項目正確性：4.3 → 3.5（**退步 0.8**）
- 三維度總平均：4.47 → 3.97

**幻覺情形**：兩模型皆無憑空捏造人名或決議，幻覺集中在三類——(1) 捏造期限：case_00 兩模型都替無期限的事項補上「本週／週內」；(2) 數字扭曲：base case_04 把備庫「一個月」寫成「一週」；(3) 負責人張冠李戴：tuned 在 case_01（沖正退款→怡萱）與 case_04（8D→Kevin）各有明確誤植。

**微調帶來的改變**：

- **精簡度**：tuned 輸出平均約 1,459 bytes，較 base 的 2,189 bytes 短約 33%，長度與行文密度明顯更貼近參考摘要風格。
- **格式穩定性**：tuned 消除了 base 的兩類輸出瑕疵——case_01 的檔尾截斷亂碼與 case_09 的模板說明殘留；標題、表格式與參考版式一致。
- **代價**：精簡的同時**丟資訊**——tuned 在 case_01、04、07 漏掉關鍵行動項目或決議，且負責人誤植比 base 多；簡體字混入反而比 base 多（3 處 vs 1 處）。

**驗收門檻判定**：

| 門檻 | 結果 |
| --- | --- |
| 結構合規 ≥9/10 | ✅ 通過（base、tuned 皆 10/10） |
| 無簡體字 | ⚠️ 程式檢查通過，但人工複核兩模型皆有漏網簡體字（base 1 處、tuned 3 處），嚴格標準下不通過 |
| 質性平均不低於基底 | ❌ **未通過**（tuned 3.97 < base 4.47；忠實度持平，要點覆蓋與行動項目正確性均低於基底） |

**建議**：目前 adapter 學到了目標風格與精簡度，但以犧牲要點覆蓋與行動項目完整性為代價，尚不宜取代基底流程。後續可 (1) 在訓練資料中強化「行動項目逐項對齊逐字稿」的樣本、(2) 修補 `SIMPLIFIED_CHARS` 字表（至少補「晓、拟、择」）或改用 OpenCC 檢測、(3) 針對負責人／期限做規則式後驗證再上線。
